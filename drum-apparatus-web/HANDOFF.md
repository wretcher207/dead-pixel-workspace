# Dead Pixel Drum Apparatus — Session Handoff

## What This Is

A standalone web app that ports the Dead Pixel Drum Apparatus REAPER Lua script (v1.1.0) to the browser. Metal producers configure drum patterns and download .mid files. Dead Pixel Design branded product.

**Source script:** `dead-pixel-drum-apparatus.lua` (project root)

## Stack

- **Svelte 5** (runes syntax) + **Vite** — chosen over Next.js because this is a pure client-side tool with no server/routing needs
- **Tailwind CSS** (v4 with Vite plugin, no separate config file)
- **Vitest** for testing
- **fflate** for client-side zip generation
- **TypeScript** throughout
- Deploys as static `dist/` to **Netlify**

## Architecture

Three domain-separated Svelte stores + one pure MIDI engine module:

```
UI Layer (Svelte components)
  ├── Library Store     → 43 preset grooves + custom patterns (localStorage)
  ├── Configurator Store → active settings (pattern, map, time sig, dynamics, fill)
  └── Session Store     → queue of configured patterns for batch export
        │
        ▼
  MIDI Engine (pure JS, zero framework dependency)
  ├── pattern-parser.ts   → notation ↔ StepEvent[] conversion
  ├── velocity-engine.ts  → humanize, mode scaling, left foot strength
  ├── timing-engine.ts    → push/pull, per-step drift, caching
  ├── fill-generator.ts   → configurable tom fills with direction + crash
  ├── midi-encoder.ts     → Standard MIDI File Type 0 binary encoder
  ├── generator.ts        → orchestrates all modules per ConfiguratorState
  └── index.ts            → public API: generatePattern(), exportSingle(), exportSession()
```

### Design Rules
- **0px border-radius** everywhere — hard edges, no rounded corners
- **No 1px borders** for sectioning — tonal carving only (background color shifts)
- **Ghost border** fallback: `rgba(90, 65, 54, 0.15)` only when accessibility requires
- **Typography:** Space Grotesk (headings, tight tracking), Inter (body/labels)
- **Animate only** `transform` and `opacity`, spring easing: `cubic-bezier(0.175, 0.885, 0.32, 1.275)`
- **SVG grain texture** at 3% opacity via `.dp-grain::before` pseudo-element
- **Sliders:** Hardware-style with sunken track, brushed-metal thumb, indicator line
- **Buttons:** Active press = `translate-y-px` for tactile depress feel

## What Needs Work — Visual Polish

The design system is applied but needs a refinement pass. Review `screenshot-styled.png` and the live site for context.

### Priority Refinements

1. **Two-column layout too cramped** — left column (grid especially) needs more breathing room. Consider adjusting flex ratios or adding more padding.

2. **Groove library categories tight** — vertical spacing between category headers and groove items needs loosening.

3. **Header lacks separation** — blends into the panels below. Needs a subtle downward shadow or the layered radial gradient background to create depth distinction.

4. **Export footer too subtle** — should feel like a prominent sticky action bar. Needs more visual weight — stronger background, maybe a top shadow like the header should have a bottom one.

5. **Grid cell sizing** — cells at w-6 h-6 may be too small for comfortable clicking on wider screens. Consider making them slightly larger or responsive.

6. **Responsive stacking** — columns stack on narrow via `lg:flex-row` but hasn't been tested on actual mobile widths. Grid needs horizontal scroll verification.

7. **Radial gradient background** — App.svelte has layered radial gradients but they may be too subtle or competing with the grain texture. Verify they're visible.

### Lower Priority

8. **Custom patterns only cover kick/snare** — text notation only handles kick and snare characters. Grid handles all 28 pieces but `stepsToNotation()` / `parseNotation()` only round-trip kick/snare.

9. **Micro-copy refinement** — design system spec suggests technical language: "INITIALIZE" instead of "Clear", "ENGAGE" instead of "Start". Optional brand-voice pass.
