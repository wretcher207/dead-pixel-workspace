# I Taught Myself to Code Through Claude. Here's What That Actually Looked Like.

I didn't go to school for this. No bootcamp. No CS degree. No uncle who worked at Google. A year and a half ago I was an audio engineer making death metal records in a basement in Maine. Today I build production websites for paying clients and run a web development business called Dead Pixel Design.

The thing that got me from point A to point B was an AI. Specifically, Claude.

I know how that sounds. Stick with me.

## The Before

I'm a musician. I play in a death metal band called Wretcher. I've spent most of my adult life in front of a DAW, not an IDE. My relationship with computers was deep but narrow. I knew my way around Reaper, MIDI, Neural DSP plugins. I could make things in a recording session that most people couldn't follow. But code? HTML was about as far as it went, and even that was rusty.

I'd tried to learn before. YouTube tutorials. Got three videos into a Python course before the monotone instructor put me to sleep. The issue wasn't the material. It wasn't motivation. Traditional learning assumes you're building toward some abstract future project. I don't work that way. I need to build something real, right now, and figure it out as I go.

## The Tool

I started using Claude the way most people do. Asking questions. Getting answers. The usual chatbot thing. Then I found Claude Code, which is different. It runs in the terminal. It reads your files, writes code, runs commands, builds things alongside you.

The first time I gave it a project description and watched it scaffold out a Next.js application, create the component structure, write the styles, and explain every decision it made, something clicked. This wasn't a search engine. This was a collaborator.

I didn't learn to code and then use Claude to speed things up. I learned to code by using Claude. Every project taught me because Claude would write the code, and then I'd read it, break it, ask why it did what it did, and rebuild it. Like learning a language by having a conversation instead of memorizing verb tables.

## What Actually Happened

It wasn't smooth. Early on I'd accept whatever Claude gave me without understanding it. That bit me more than once. I'd ship something, a client would ask for a change, and I'd realize I had no idea why the code was structured the way it was.

The fix was simple but required discipline: stop copy-pasting blindly and start reading. Every function Claude wrote, I'd ask it to explain. Every architectural decision, I'd push back and ask for alternatives. Over time the balance shifted. I went from "build me a navbar" to "here's how I want the navigation state to work, implement it this way."

Here's what nobody tells you about learning through AI: it's not faster because it's easier. It's faster because you skip the artificial curriculum and go straight to building real things. You still have to put in the work. You still have to understand what's happening. The difference is you're learning by doing, not by watching someone else do it on YouTube.

## Why This Matters

Most people I talk to have tried ChatGPT, gotten underwhelming results, and written off AI entirely. I get it. I would have too if that had been my first experience.
